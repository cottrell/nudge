## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Think later: automate a short terminal demo (GIF/asciinema/VHS or stills) of the operator surface without needing real LLM agents.

## Sketch flow (illustrative)

1. `aiswarm init test` (or dedicated demo/fixture project name)
2. Point panes at dumb shells if needed (bash / echo-loop) so activity/idle is controllable — real agent TUIs optional
3. `aiswarm start ./swarm/test.yaml`
4. Split view / dual capture:
   - one pane/window: `aiswarm status ./swarm/test.yaml --brief -w` and/or `aiswarm log ./swarm/test.yaml -w` (and `--pending`)
   - other: the tmux swarm grid (agents or shell stand-ins)
5. `aiswarm send ./swarm/test.yaml 0.2 "say hello to pane 0.2"` (or broadcast --via-log)
6. Watch log show the event; watch pending clear / consumer deliver via tmux-send when pane is idle
7. Optional: poke pane activity so working→idle is visible
8. Teardown: `aiswarm stop ./swarm/test.yaml` (and kill recorders)

## Goals

- Honest demo of status + durable log comms + delivery-on-idle
- Deterministic / re-recordable for README
- Safe (no secrets, no live model auth required for v1)
- Prefer synthetic panes over real Claude/Codex unless we want TUI chrome later

## Design questions

- Recorder: VHS, asciinema+agg, tmux capture-pane stills, or ffmpeg of a real terminal?
- Single script (`make demo` / `scripts/demo-capture.sh`) vs documented manual checklist
- Demo YAML in-repo (`examples/` or generated by init) with stable pane layout
- How to show dual views (status/log vs grid) cleanly in one recording
- Whether babysit/quota pacing is in-scope for v1 (probably not)
- CI vs on-demand only (likely on-demand; commit artifact or regenerate on release)

## Out of scope for first think-through

- Fully automated live multi-model swarm movie
- Perfect GIF polish / marketing edit
